# 净排 UX 架构重设计 · 概览

**日期**：2026-07-20 ｜ **角色**：用户体验架构师（ArchitectUX）

## 做了什么
梳理了净排（v0.13.3）全链路 UX，定位 12 处体验断点，并给出围绕「写作→排版→预览→发布」生命周期的流畅架构重设计。

## 关键结论
- **最大转化断点 B4**：复制→微信后台 是断崖。解法 = 「发布向导」三步闭环（复制 → 一键开微信 → 粘贴自检清单）。
- **一致性断点 B1/B5**：装饰面板残留「图片」概念；样式面板与品牌色两处割裂。解法 = 装饰纯净化 + 「外观」三层显式级联（主题→品牌→微调）。
- **首屏冲突 B9**：恢复横幅/引导/空状态 无优先级。解法 = 入口协调器（恢复 > 引导 > 空状态）。
- **架构根因 B10**：`App.vue` ≈ 2800 行单组件。解法 = 按组件树拆分 + `tokens.css` 设计基座。

## 交付物
- 完整文档：`docs/UX架构重设计-断点与流畅架构.md`（断点清单 + 重设计架构 + 实施/验收清单 + Before/After 流程）
- 12 个断点按 高/中/低 分级，实施清单分 P0/P1/P2 + 架构根因，均对齐既有 PRD/ADR/RELEASE 流程。

## 下一步
