## 第1步：PreviewPanel

### 拆什么

**模板**（App.vue 当前行号范围）：
```

<section class="preview-section" v-show="showPreview">
  <div class="phone-frame" :class="phoneModel">
    <div class="phone-wechat-bar">...</div>
    <div class="phone-content" v-html="previewHtml"></div>
```

```vue

<script setup>
defineProps({
  showPreview: Boolean,
  previewHtml: String,
  phoneModel: String,
  previewWidth: Number,
})
</script>
```

**无 emit** —— 预览区是纯展示组件。手机型号切换通过 `v-model:phoneModel` 双向绑定。

### 父组件调用

```vue

</script>
<template>
  <PreviewPanel
    :show-preview="showPreview"

  />
</template>
```

## 第2步：AIPanel

### 拆什么

**模板**（App.vue 中 `v-show="activePanel === 'ai'"` 的整个 `<div class="floating-panel">`）：

```html
<div class="panel-header"><h3>AI 写作</h3>...</div>
<div class="panel-content">
  <!-- 提示文案 -->
  <!-- 接口预设下拉 -->
  <!-- API Key 输入 -->
  <!-- Base URL 输入（条件） -->
  <!-- 模型名输入 -->
  <!-- 场景化推荐按钮 + 更多操作折叠 -->
  <!-- 加载行 -->
</div>
```

### 组件接口

```vue
<!-- src/components/AIPanel.vue -->
<script setup>
import { ref } from 'vue'

const props = defineProps({
  // AI 配置（v-model）
  aiPreset: String,
  aiKey: String,
  aiBaseUrl: String,
  aiModel: String,
  // 状态
  aiLoading: Boolean,
  aiRecommendations: Object, // { primary: [], rest: [] }
})

const emit = defineEmits([
  'update:aiPreset', 'update:aiKey', 'update:aiBaseUrl', 'update:aiModel',
  'runAction',  // payload: 'generateTitle'|'writeSummary'|'expand'|'structure'
  'close',
])
</script>
```

### 父组件调用

```vue
<AIPanel
  v-model:ai-preset="aiPreset"
  v-model:ai-key="aiKey"
  v-model:ai-base-url="aiBaseUrl"
  v-model:ai-model="aiModel"
  :ai-loading="aiLoading"
  :ai-recommendations="aiRecommendations"
  @run-action="runAiAction"
  @close="activePanel = null"
/>
```

**不需要 emit 的**：`onAiPresetChange` / `saveAiConfig` / `cancelAI` —— 这些函数保留在 App.vue，由 `@update:aiPreset` 等 v-model 事件触发 watch。

### AIPanel 模板中保留的内部状态

| 状态 | 说明 |
|------|------|
| `<details>` 展开/折叠 | 纯 CSS/HTML，无 JS 状态 |
| 接口预设切换后 `v-if="aiPreset === 'custom'"` 的条件显示 | 由 props 驱动 |

## 第3步（上）：useStyle.js composable

### 为什么先拆 composable 再拆模板

StylePanel 模板依赖 13 个 props + 6 个事件，如果直接塞给子组件，props 会多到难以维护。先把样式逻辑抽成 `useStyle.js`，让 App.vue 和 StylePanel 共享同一个 composable 的返回值，模板拆分时只需要传 `useStyle` 返回的对象即可。

### 搬什么

从 App.vue 中**完整复制**以下函数到 `useStyle.js`，逻辑一行不改：

| 函数 | 当前行号 | 说明 |
|------|---------|------|
| `getDefaultStyle(tag, prop)` | L1044 | 获取标签的默认样式值 |
| `loadTagStyle()` | L1053 | 从 customOverrides 加载当前标签样式到面板滑块 |
| `applyCustomStyle()` | L1070 | 将滑块值写入 customOverrides + 触发 convertMarkdown |
| `resetTagStyle()` | L1090 | 重置当前标签回默认值 |
| `applyThemeToAll()` | L1100 | 一键套用整套主题到 customOverrides |

**不搬的**：`selectTheme`（涉及 `currentTheme` 和 `onThemeChange`，留在 App.vue 因为它会触发 `convertMarkdown` 和 `saveSettings`）。

### 接口

```js
// src/composables/useStyle.js
export function useStyle(customOverrides, selectedTag, 
  tempFontSize, tempColor, tempLineHeight, tempLetterSpacing,
  convertMarkdown, showToast) {
  
  function getDefaultStyle(tag, prop) { /* 搬家 */ }
  function loadTagStyle() { /* 搬家 */ }
  function applyCustomStyle() { /* 搬家 */ }
  function resetTagStyle() { /* 搬家 */ }
  function applyThemeToAll(currentTheme, THEMES) { /* 搬家 */ }

  return {
    getDefaultStyle, loadTagStyle, applyCustomStyle,
    resetTagStyle, applyThemeToAll,
  }
}
```

## 第3步（下）：StylePanel

### 拆什么

**模板**（App.vue 中 `v-show="activePanel === 'style'"` 的整个 `<div class="floating-panel">`，约 80 行）

### 组件接口

```vue
<!-- src/components/StylePanel.vue -->
<script setup>
import { ref } from 'vue'
import { THEME_NAMES } from '../themes.js'

const props = defineProps({
  currentTheme: String,
  themes: Object,          // THEMES 常量
  selectedTag: String,
  tempFontSize: Number,
  tempColor: String,
  tempLineHeight: Number,
  tempLetterSpacing: Number,
  themeUiAccents: Object,  // { key: accentColor } 预计算
})

const emit = defineEmits([
  'update:selectedTag',
  'update:tempFontSize', 'update:tempColor',
  'update:tempLineHeight', 'update:tempLetterSpacing',
  'selectTheme',
  'loadTagStyle',
  'applyStyle',
  'applyAll',
  'resetTag',
  'close',
])

// 主题下拉展开/收起（组件内部 UI 状态）
const showDropdown = ref(false)
function toggleDropdown() { showDropdown.value = !showDropdown.value }
</script>
```

### 父组件调用

```vue
<StylePanel
  :current-theme="currentTheme"
  :themes="THEMES"
  :theme-ui-accents="themeAccents"
  v-model:selected-tag="selectedTag"
  v-model:temp-font-size="tempFontSize"
  v-model:temp-color="tempColor"
  v-model:temp-line-height="tempLineHeight"
  v-model:temp-letter-spacing="tempLetterSpacing"
  @select-theme="selectTheme"
  @load-tag-style="loadTagStyle"
  @apply-style="applyCustomStyle"
  @apply-all="applyThemeToAll"
  @reset-tag="resetTagStyle"
  @close="activePanel = null"
/>
```
