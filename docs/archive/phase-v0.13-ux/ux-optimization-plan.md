## 二、已确认的两个痛点（根因）

### 1. 图片插入只能填 URL
- 位置：`App.vue` **L344–353** 的面板只有一个 `imageUrl` 文本框 + `插入图片` 按钮；`insertImageByUrl`（L1596）仅支持 URL。
- 缺失：本地文件选择、剪贴板粘贴、拖拽上传。
- **额外约束（关键）**：微信后台会过滤 `<img src="data:...">`（见 CHANGELOG v0.7.0）。所以本地图片即便能插入编辑器/本地预览，复制进微信也会丢。这是个**产品策略问题，必须先定方案**。

### 2. 深色模式文字/背景糊在一起
- 根因：v0.10.0 定了「文章正文恒白底、深色只作用于外壳」，但这个决策**只做了一半**：
  - 预览区 `.phone-content`（L2501）没有显式白底，深色下透出外壳深色背景 → 深色文字落在深色背景上，几乎看不见。
  - 编辑器 `.milkdown .ProseMirror`（L3248）的文字色被强制成外壳的浅色 `var(--text-primary)`，而编辑区底是白的 → 浅色字落在白底上，同样难读。
- 两处都属于「文章白底没钉死 / 文字色被外壳 token 带偏」。
