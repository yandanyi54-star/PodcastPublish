# 净排 · 用户体验架构重设计（流畅版）

> 角色：用户体验架构师（ArchitectUX）
> 日期：2026-07-20
> 基线：`v0.13.3`，`src/App.vue` ≈ 2800 行（单组件承载全部 UI + 逻辑）
> 目标：梳理产品架构、定位体验断点、给出更流畅的 UX 架构与可落地清单
> 硬约束（沿用）：纯前端 · 不登录 · 内容不入库 · 不联网（AI 仅用户主动触发的浏览器直连）

---

## 0. 一句话结论

净排的核心价值链是 **写作 → 排版 → 预览 → 发布**，断点几乎全部发生在「阶段交界」与「被移除功能留下的残留」上。其中 **复制→微信后台的发布断层** 是产品最大的转化断点（也是该品类通病），**装饰/图片概念残留** 与 **样式面板与品牌色两处割裂** 是最易引发困惑的一致性断点。重设计的重点是：把「发布」做成有引导的闭环、把「样式」收敛成显式级联、用「入口协调器」消除三重首屏冲突，并以设计系统基座根治 2800 行单文件带来的体验漂移。

## 1. 现状用户流程地图

```
[进入]
  ├─ 恢复横幅?  ──┐
  ├─ Onboarding? ─┤  ← 二者可能同时出现，无明确优先级（断点 B9）
  └─ 空状态指引  ──┘
        ↓
[写作] 编辑器 ← 导入（提纯后无明确去向）/ 撤销·重做
        ↓
[排版] 样式面板（10 主题 + 按标签微调）
        + 设置面板（品牌色 + 品牌字体）   ← 两处样式，边界模糊（断点 B5）
        + 装饰面板（残留「图片」概念：标签/tooltip/尺寸文案）（断点 B1）
        ↓
[预览] 实时手机预览
        + 「微信深色」开关（却被放在「发布」工具栏组）（断点 B2）
        ↓
[发布] 复制 HTML → 仅弹 6 秒 toast → 用户自行开微信  ← 最大断层（断点 B4）
```

可见：入口三选一冲突、写作→排版有两套样式心智、排版→预览混入了发布概念、预览→发布是断崖。

## 2. 断点清单（按严重度）

| 编号 | 位置 | 现象（代码/行为证据） | 影响 | 严重度 |
|------|------|----------------------|------|--------|
| **B1** | 左侧工具栏「装饰」项 + 装饰面板 | 工具栏 `activePanel==='image'`、tooltip「装饰 / 图片」；面板标题「装饰」却含「📷 如需配图，请写 `![描述](图片链接)`」；装饰卡 `decor-meta` 写「800×400 · 顶部展示」（图片尺寸）。图片功能 v0.13.1 已移除，但 UI 残留。 | 用户预期「能插图」，心智与事实不符；装饰面板语义不纯。 | 🔴 高 |
| **B2** | 工具栏「发布」组 | 「微信深色」开关与 预览/复制/导出 并列于发布组，但它**仅影响预览**（`previewWechatDark`）。 | 发布组内混入预览概念，开关用途不透明。 | 🟡 中 |
| **B3** | 外观模式控件 | `cycleThemeMode` 每次点击循环 light→dark→system，仅靠文字标签显状态；spec-ux 文档本要求三态分段 + `aria-pressed`。 | 无法直达「跟随系统」；当前态不直观；可访问性差。 | 🟡 中 |
| **B4** | 复制 → 微信后台 | 工具栏「复制」只复制 + toast；`openWeChat` 仅藏在导出面板；无步骤化引导与粘贴自检。 | 发布是产品存在的理由，却是最弱一环——用户被丢回净排，无下一步。 | 🔴 高 |
| **B5** | 样式面板 vs 设置面板 | 「样式」(10 主题 + 按标签微调) 与「设置」(品牌色+字体) 两处独立；品牌色只重染强调色、`applyThemeToAll` 清空微调但不动品牌（正交关系未对用户说明）。 | 两处都能「改外观」→ 决策瘫痪；不懂为何改品牌色不动全局、为何一键应用会清空微调。 | 🟡 中 |
| **B6** | 编辑器上下文栏 | 标题选中恒显示；文字选中且 ≥10 字才显示 AI 操作；<10 字时栏显「文字(N字)」却**无任何操作**→死路。 | 栏出现但不给操作，用户困惑「它干嘛」。 | 🟡 中 |
| **B7** | 导入面板 | 「从文本提纯」后无确认、无去向下一步；与空状态「载入示例文章」竞争。 | 导入后用户不知道「成功了吗、然后呢」。 | 🟡 中 |
| **B8** | 响应式三档 | 断点 768（左栏→底栏、预览→全屏覆盖）与 1024（面板收窄）。769–1024 仍桌面分屏 + 面板 push 280px + 预览 38vw，iPad 竖屏(810)编辑器被挤压。 | 平板区间编辑器过窄、面板推挤。 | 🟡 中 |
| **B9** | 首屏入口 | 恢复横幅 / Onboarding(4 步+载示例) / 空状态 三者无优先级裁决；引导第 4 步「开始创作」会载入示例，可能覆盖已恢复草稿。 | 首屏提示冲突或意外覆盖草稿。 | 🟡 中 |
| **B10** | 架构根因 | `App.vue` ≈ 2800 行单组件。B1/B6 等不一致本质是「无组件边界强制统一 UX 模式」。 | 体验断点会持续累积、难以根治。 | 🔴 高（根因） |
| **B11** | AI 面板 | Onboarding 把「AI 写作」列为功能，但 AI 需自备 Key；首次打开无前置说明。 | 首次用 AI 像「坏了」。 | 🟢 低 |
| **B12** | 复制 toast | toast 警告「图片为本地 base64，微信会替换为…」，但图片功能已移除（现仅 `![alt](url)` 外链）；对用外链图的用户是误导。 | 无谓告警，甚至吓到正常用户。 | 🟢 低 |

## 3. 重设计的流畅 UX 架构

### 3.1 信息架构重排（围绕生命周期，而非实现）

工具栏从「按功能堆」改为「按 4 阶段 + 系统」分区，且让每个分区语义自洽：

```
写作区   撤销 · 重做 · 导入
排版区   外观(主题/品牌/微调) · 装饰
预览区   预览开关 · 微信深色(预览模拟) · 阅读信息
发布区   复制 · 导出 · 打开微信(闭环入口)
系统区   设置 · 外观模式(三态) · 快捷键帮助
```

关键归位：**微信深色 → 预览区**（B2）；**品牌色 → 并入「外观」**（B5）；**打开微信 → 发布区**（B4）。

### 3.2 发布闭环（最大转化断点的解法）—「发布向导」

把一次性 toast 升级为可一步完成的 3 步引导（不强制弹窗，复制后给常驻轻引导）：

```
① 复制 HTML      → 剪贴板确认（成功态，非 6 秒警告）
② 打开微信后台  → 一键 window.open(mp.weixin.qq.com)（从导出面板提到主发布流）
③ 粘贴自检清单  → 封面/分割线/金句 自动生效；
                  外链图正常、base64 本地图需素材库上传；
                  提醒读者深色端观感（呼应 v0.13.3 微信深色模拟）
```

这是产品价值兑现点，必须做成一等公民，而非隐藏在导出面板。

### 3.3 样式模型统一（显式级联，消除两处割裂）

把「样式」与「设置-品牌」合并为单一「外观」表面，明确三层叠加关系：

```
Layer 1  主题      一键换肤（10 套），影响全局配色/字体/间距
Layer 2  品牌色    可选开关：仅重染强调色（h1/a/引用），保留主题骨架
Layer 3  微调      可选：按标签(字号/颜色/行距/字距)精修
级联说明：品牌色叠加在主题之上；「一键应用」会清空 Layer 3 微调但保留 Layer 2 品牌
```

把「为什么改品牌色不动全局」「为什么一键应用清空微调」直接写进面板提示，消除 B5 困惑。

### 3.4 入口协调器（消除首屏三选一冲突）

单一裁决源，优先级明确：

```
1. 有恢复草稿      → 仅显示恢复横幅；绝不自动载示例、绝不触发 Onboarding 覆盖
2. 真·首次（无草稿且无 onboardingDone） → 显示 Onboarding（结束即进入空编辑态，不强制载示例）
3. 其余            → 编辑器空状态内联「载入示例文章」入口
```

Onboarding 的「开始创作」不再静默覆盖现有内容。

### 3.5 上下文操作一致性

- 上下文栏**仅在有 ≥1 个可用操作时出现**。
- 短文（<10 字）选中：给格式化操作（加粗/斜体/链接）而非空白。
- 装饰块/标题：保留各自操作。杜绝 B6 死路。

### 3.6 响应式三档策略（显式、保护编辑器）

```
≤768    移动端：底部标签栏 + 预览全屏覆盖层 + FAB 切换；面板全宽覆盖
769–1024 平板：面板改为「覆盖」而非「push」，预览侧栏但保证编辑器最小宽 ≥480px
≥1025   桌面：编辑器/预览分屏 + 面板 push
```

核心原则：**任何断点下编辑器可用宽度 ≥ 480px**，面板不得挤压正文（解决 B8）。

### 3.7 外观三态控件（取代循环）

用分段三态控件 + `aria-pressed`，状态一目了然、可直达：

```html
<div class="theme-mode-switch" role="radiogroup" aria-label="外观模式">
  <button role="radio" aria-checked="false" data-mode="light">浅色</button>
  <button role="radio" aria-checked="false" data-mode="dark">深色</button>
  <button role="radio" aria-checked="true"  data-mode="system">跟随</button>
</div>
```

### 3.8 装饰与图片解耦

二选一（建议 A，符合「纯本地」与微信不存 base64 的现实）：
- **A（推荐）**：工具栏与面板统一命名为「装饰」，彻底删除「图片」字样与「800×400」等图片尺寸文案；图片仅以 `![alt](url)` 外链形式由用户自行写入，并在帮助中说明「微信后台对 base64 本地图不友好」。
- **B**：若确认要保留「插图(URL)」能力，则给一个**独立、诚实**的「插入图片(URL)」入口，明确标注「仅外链、非本地上传」，与装饰并列而非混用。

### 3.9 设计系统基座（ArchitectUX 交付物，根治 B10）

建立 `src/styles/tokens.css` 设计变量基座，使组件拆分后不漂移：

```css
:root {
  /* 语义色（亮色） */
  --bg-primary:#ffffff; --bg-secondary:#f6f5f3; --bg-hover:#ece9f2; --bg-active:#e2ddef;
  --text-primary:#1f1d29; --text-secondary:#6b6878; --text-tertiary:#9b98a8;
  --border-light:#e7e4ee; --border-medium:#d6d2e2;
  --accent:#07c160;            /* 品牌绿，随主题覆盖 */
  --shadow-sm:0 1px 2px rgba(20,18,30,.06);
  --radius-sm:8px; --radius-md:12px; --radius-lg:16px;
  --space-1:4px; --space-2:8px; --space-3:12px; --space-4:16px; --space-6:24px; --space-8:32px;
  --transition-fast:120ms ease; --transition-normal:200ms ease;
}
[data-theme="dark"] { /* 派生深色全集，对比度 ≥4.5:1 */ }
@media (prefers-color-scheme: dark){ :root[data-theme="system"]{ /* 套用 dark */ } }
```

建议组件树（承接 v0.13.0 已开始的拆分）：

```
src/
├─ App.vue                      编排 + 入口协调器
├─ components/
│  ├─ EntryCoordinator.vue      恢复横幅/引导/空状态 优先级裁决
│  ├─ toolbar/
│  │  ├─ AppToolbar.vue         5 区布局
│  │  ├─ ThemeModeSwitch.vue    三态分段（3.7）
│  │  └─ ToolbarGroup.vue
│  ├─ panels/
│  │  ├─ AppearancePanel.vue    主题 + 品牌 + 微调（统一 3.3）
│  │  ├─ DecorPanel.vue         纯装饰（3.8）
│  │  ├─ ImportPanel.vue        导入 + 明确去向（3.5/B7）
│  │  ├─ AIPanel.vue
│  │  ├─ ExportPanel.vue
│  │  └─ PublishWizard.vue      发布闭环（3.2）
│  ├─ EditorArea.vue            编辑器 + 上下文栏 + 状态栏
│  └─ PreviewPanel.vue
├─ composables/                 useDraft / useBrand / useStyle（已有）
└─ styles/
   ├─ tokens.css                设计变量基座
   ├─ themes.css                亮/暗/跟随
   └─ components.css
```

### 3.10 无障碍基线

- 主题切换 `radiogroup` + `aria-checked`；面板/浮层 `role="dialog"` + `aria-modal` + Esc 关闭 + 焦点陷阱。
- 工具栏全部 `role="button"` + `aria-label`（现状已有，保持）。
- 深色模式全量 token 审计，文字/图标对比度 ≥ WCAG 2.1 AA。
- 键盘可达：工具栏 Tab 顺序与视觉分区一致；面板内可 Tab 操作。

## 附录：Before / After 流程对照

**Before（有断点）**
```
进入 → 恢复横幅?+引导?+空状态?（可能同屏冲突）
写作 → 编辑器 / 导入(无去向)
```

**After（流畅）**
```

排版 → 统一「外观」: 主题 → 品牌 → 微调（显式级联） / 纯装饰
预览 → 预览开关 + 微信深色(归位) + 阅读信息
发布 → 发布向导: ①复制 ②开微信 ③自检清单（闭环）
```
