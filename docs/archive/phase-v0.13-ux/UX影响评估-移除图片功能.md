# UX 影响评估：移除图片功能

> 日期：2026-07-20
> 视角：净排用户（公众号作者）的完整排版工作流

---

## 一、当前状态：图片功能实际表现（诚实评估）

| 操作 | 用户预期 | 实际发生 |
|------|---------|---------|
| 拖入图片 | 图片出现在编辑器中 | ✅ 出现了 |
| 点击图片调宽度 | 宽度变化 | ❌ 弹「检测不到该图片」 |
| 删除图片 | 图片消失 | ❌ 弹「检测不到该图片」 |
| 替换图片 | 图片被换掉 | ❌ 弹「检测不到该图片」 |
| 调完宽度看预览 | 预览中宽度变化 | ⚠️ 预览变了，编辑器没变 |
| Ctrl+Z 撤销插入 | 图片消失 | ⚠️ 有时能撤销，有时留下 `---` |
| 复制 HTML 到微信 | 图片正常显示 | ❌ 微信把图片换成 `…` |
| 用户心态 | 「这个工具能帮我排版」 | 「这个工具图片功能是坏的」 |

**结论**：当前图片功能给用户的体验是**负面的**——它的存在让用户觉得产品不稳定，而不是提供价值。

---

## 二、移除后的用户工作流变化

### 之前的流程（理想情况）
```
净排写文章 → 拖图片进去 → 调宽度 → 复制到微信 → 发布
                  ↑（已坏）    ↑（已坏）   ↑（微信不支持）
```

### 移除后的流程
```
净排写文章 → 复制到微信 → 在微信编辑器中插入图片 → 发布
```

### Markdown 高手流程（保持不变）
```
净排写文章 → 手写 ![alt](http://...) → 预览中看到图 → 复制到微信 → 发布
```

## 三、按用户画像的影响分析

### 画像 A：文字为主、偶尔配图
**占比**：最多
**影响**：🟢 低
- 本来就只用文字 + 装饰元素排版
- 配图在微信后台手动加
- 移除图片功能后体验不变

### 画像 B：重度图片使用者
**占比**：少
**影响**：🔴 高，但当前图片功能对这部分用户的体验是「用不了」
- 与其给一个坏功能，不如没有
- 可以引导去微信后台加图（这是微信用户的常规习惯）

### 画像 C：Markdown 高手
**占比**：少
**影响**：🟢 无
- 手写 `![alt](url)` 图片仍渲染
- 从未用过拖拽/编辑面板

---

## 四、替代方案：装饰面板的 UI 改造

当前装饰面板的「图片上传区」替换为一句引导文案：

```html
<div class="image-notice" style="text-align:center;padding:16px;color:var(--text-tertiary);">
  <p style="margin-bottom:8px;">📷 图片功能已移除</p>
  <p style="font-size:13px;">
    排版完成后，复制 HTML 到微信后台，再上传图片。<br/>
    或使用 Markdown <code>![描述](图片链接)</code> 直接引用网络图片。
  </p>
</div>
```

**好处**：
1. 不留下空白面板区域
2. 主动告知后续操作路径
3. 教用户「先在净排排版，再到微信加图」的正确习惯

---

## 五、净益评估

| 维度 | 移除前 | 移除后 | 变化 |
|------|--------|--------|------|
| 用户是否可以插图片 | ⚠️ 可以，但大概率失败 | ❌ 不可以 | 功能移除 |
| 用户是否遇到 Bug | 🔴 每次图片操作都遇 Bug | 🟢 永远不会 | Bug 清零 |
| 用户信任感 | 🔴「这工具不稳定」 | 🟢「这工具靠谱」 | 信任提升 |
| 用户学习成本 | ⚠️ 需要学会忍受 Bug | 🟢 一句话引导 | 成本降低 |
| 装饰面板整洁度 | 🟡 混杂图片功能 | 🟢 纯净排版 | 界面提升 |

---

## 六、建议

**移除图片功能** → 在装饰面板留一句引导文案 → 把 UI 留给真正稳定的功能（装饰元素）。

删除 4 个 Bug 换来一个干净的界面 + 用户信任 + 零回归风险。值。
