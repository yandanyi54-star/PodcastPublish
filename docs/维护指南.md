# 净排 · 维护指南（给非工程背景的 owner）

> 目的：让你以后想改点东西、发个新版，不用回来看代码也能照着做。
> 适用版本：v0.14.x 起。技术栈：Vue 3 + Vite + Milkdown（所见即所得编辑器），纯前端、不登录、内容不上云。

---

## 0. 一句话心智模型

净排 = 一份网页（`dist/` 文件夹里的 3 个文件）。
- **改功能/文案** → 改 `src/` 里的源代码 → 运行 `npm run release` 重新生成 `dist/`。
- **发上线** → 把 `dist/` 推到两个地方：
  - 全球版：GitHub Pages（`https://yandanyi54-star.github.io/jingpai/`）
  - 国内版（给 HR / 招聘官）：CloudStudio 静态空间（国内打开快）

你**不需要懂代码**也能做日常调整（主题、配色、AI 预设都在配置里）；只有加功能才动逻辑。

---

## 1. 环境准备（首次或换电脑时做一次）

需要 Node.js（建议 22，或 18+）。装好后在项目文件夹里打开终端：

```bash
npm install        # 安装依赖（只需一次）
```

日常命令（记住这 3 条就够）：

| 命令 | 作用 |
|------|------|
| `npm run dev` | 本地预览，浏览器开 `http://localhost:3000`，改代码实时生效 |
| `npm run release` | **发版前必跑**：自动跑「代码检查 + 测试 + 构建」，全绿才产出新的 `dist/` |
| `npm run check` | 只做「检查 + 测试 + 构建」的安全网，不改任何东西，用来确认没改坏 |

> 💡 每次改完任何东西，跑一次 `npm run release`，全绿再发布。这是你的"自动兜底"。

## 2. 「我想改 X」→ 改哪个文件（查表）

| 你想改的东西 | 改哪里 | 难度 |
|------|------|------|
| **主题配色 / 字体**（10 套主题） | `src/themes.js` | 低（改颜色值） |
| **新增一套主题** | `src/themes.js`（见下方步骤） | 低 |
| **AI 预设 / 默认模型**（DeepSeek 等） | `src/App.vue` 里的 `AI_PRESETS` + `src/components/AIPanel.vue` 下拉项 | 低 |
| **界面上的按钮 / 提示语 / 引导文案** | `src/App.vue`（散落各处，搜中文关键词即可定位） | 低 |
| **品牌色 / 正文字体**（单篇重染） | 软件界面内「外观」面板，**不用改代码** | 零 |
| **装饰语法**（封面/分隔/金句） | 写作时用 `:::cover` 等语法（见第 4 节红线） | 零 |

### 2.1 改现有主题配色
打开 `src/themes.js`，每套主题是一组 CSS 片段，例如：
```js
serif_news: {
  body: "background:#f9f7f1; ...",      // 正文背景 + 字体
  h1:   'font-size:32px;color:#8b0000; ...', // 一级标题颜色
  a:    'color:#8b0000; ...',           // 链接颜色
  uiAccent: '#b57373'                   // 界面强调色
}
```
**只改颜色值（如 `#8b0000`）即可**，别动字段名（body/h1/a…）。改完跑 `npm run release`。

### 2.2 新增一套主题（3 步）
1. 在 `src/themes.js` 顶部 `THEME_NAMES` 加一行：`my_theme: '我的主题'`。
2. 在 `THEMES` 里照抄任意一套（如 `serif_news`），把 key 改成 `my_theme`，改里面的颜色。
   - **必须包含全部 25 个字段**（body / h1~h6 / p / blockquote / ul / ol / li / img / a / code / pre / hr / table / th / td / strong / em / del / mark / uiAccent），缺一会排版错位。
3. 跑 `npm run release` → 外观面板里就能选到新主题。

### 2.3 改 AI 预设 / 默认模型
- 预设列表在 `src/App.vue` 的 `AI_PRESETS`（约 595 行）：
  ```js
  const AI_PRESETS = {
    deepseek:  { baseURL: 'https://api.deepseek.com/v1',   model: 'deepseek-chat' },
    openai:    { baseURL: 'https://api.openai.com/v1',      model: 'gpt-4o-mini' },
    openrouter:{ baseURL: 'https://openrouter.ai/api/v1',   model: 'openai/gpt-4o-mini' },
    custom:    { baseURL: '', model: '' }
  };
  ```
  加新厂商：照格式加一行（填它的 Base URL 和模型名）。
- 默认模型：`const aiPreset = ref('deepseek')` 改引号里的 key 即可换默认。
- 下拉框显示项在 `src/components/AIPanel.vue`（`<option value="deepseek">DeepSeek</option>`），**加了预设记得这里也加一行**，否则界面选不到。

## 3. 发版流程（每次更新都走这套）

1. 改完代码/配置。
2. 跑 `npm run release`（自动检查+测试+构建，全绿继续；红了先修再继续）。
3. 升版本号：改 `package.json` 里的 `"version"`（如 `0.14.1` → `0.14.2`）。
4. 写更新说明：在 `CHANGELOG.md` 顶部加一行本次改了啥。
5. 发布（见第 4 节）。
6. **提交源码**：`git add . && git commit -m "v0.14.2 改了XX"`（源码存到 git，方便回滚）。

## 4. 部署（两个版本）

> 当前 `dist/` 用相对路径构建（`vite.config.js` 的 `base: './'`），**同一份构建国内外都能直接用**，不用为不同托管改配置。

### 4.1 全球版 · GitHub Pages
```bash
npm run deploy:github     # 把 dist/ 推到仓库的 gh-pages 分支
```
首次需要去 GitHub 仓库 **Settings → Pages → Source 选 `Deploy from a branch` → 选 `gh-pages` / `/root` → Save**（只点一次，之后自动生效）。
上线地址：`https://yandanyi54-star.github.io/jingpai/`

### 4.2 国内版 · CloudStudio（给 HR / 招聘官）
让 WorkBuddy 执行「部署到 CloudStudio」（或对话里说"把净排发到 CloudStudio"），会把 `dist/` 传到国内静态空间，返回一个国内打开快的链接。适合放在简历 / 作品集里给招聘方点。

> 两个版本内容完全一致，只是托管地不同。建议：GitHub 版放技术细节，CloudStudio 国内版放简历链接。

## 5. 脆弱区红线（改之前务必看，别碰这些）

这些地方历史上踩过坑，**动错会静默失效**，非必要不要改：

- 🔴 **装饰语法必须无空格**：写作时用 `:::cover`、`:::divider`、`:::quote`（冒号后**直接跟类型名，不能空格**）。写成 `::: cover` 会被当成普通文字不解析。
- 🔴 **Milkdown 插件**（`src/plugins/`）和编辑器 schema 顺序：牵一发动全身，别随便改。
- 🔴 **localStorage 的 key 约定**：`podcast_draft` / `podcast_settings` / `podcast_theme_overrides` / `podcast_brand` / `podcast_ai_config`。这些是用户数据存档的"门牌号"，改名会导致用户草稿/设置丢失。
- 🟡 `vite.config.js` 的 `base: './'` 不要改回 `/`，否则 GitHub 子路径下资源 404。
- 🟡 主题字段必须 25 个全齐（见 2.2）。

## 6. 回滚
所有源码在 git 里。改坏了就：
```bash
git log --oneline      # 看历史
git checkout <某次commit的号> -- src/某个文件   # 只恢复某个文件
# 或 git revert <commit号> 整笔撤回
```
恢复后跑 `npm run release` 重新构建、再 `deploy:github` 重新发。
