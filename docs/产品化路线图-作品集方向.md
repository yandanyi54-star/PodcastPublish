# 净排 · 产品化路线图（作品集方向 · 产品岗校准版）

> 校准日期：2026-07-22
> 关键变化：作者为文科生，目标 = **产品岗实习 / 面试**；净排是作品集，且作者自己用（有公众号）。受众 = **产品招聘官**，不是工程师。

---

## 0. 重新审视：展示给谁看

- 旧假设：受众是工程师，靠技术深文 + GitHub stars。
- 新现实：受众是**产品招聘官**；他们看的是**产品思维、决策逻辑、执行力、真实用户视角**，不是代码。
- 因此展示重心从"技术深度"转向"**产品叙事 + 可体验的产物**"。

## 1. 最优先：一个"方便展示"的载体

- [ ] **产品案例文档（核心交付）**：`docs/净排-产品案例.md`——给招聘官的项目复盘，展示 痛点 → 洞察 → 竞品 → 决策 → 迭代 → 复盘。可导出 PDF 随简历。
- [ ] **Live demo 部署**：GitHub Pages / Vercel，README 顶部放链接。让人 2 分钟试用。
- [ ] **README 加"产品故事"入口**：在代码仓库里也放案例文档链接，工程师 / 招聘官都能进。

## 2. 内容策略（文科生友好、产品向）

不再写 Milkdown 技术深文（那是工程师叙事）。改为：

- [ ] **A1 竞品分析文**：mdnice / Plainly / 秀米 对比 + 净排差异化（发自己公众号 + 知乎）。
- [ ] **A2 产品决策文**：为什么导出恒定浅色、预览加微信深色模拟（展示"基于事实决策"）。
- [ ] **A3 公众号复盘**：用净排排自己的文章，写"一个文科生怎么做产品工具"（dogfooding 实证，直接发自己公众号）。
- 这些既是公众号内容（养号），又是作品集素材，一举两得。

## 3. 社区与分发

- [ ] V2EX「分享创造」/ 即刻：用**产品视角**讲历程，不是技术细节。
- [ ] 导航站：少数派 / 利器 / **AlternativeTo**（打 mdnice 替代品）。
- [ ] 自己的公众号是天然分发渠道 + 实证。

## 4. 展示闭环

- [ ] 案例文档 PDF 版（投递用）。
- [ ] Live demo 常驻可用。
- [ ] 公众号持续用净排发文章（"我做的工具排版了我自己的号"）。

## 5. 明确不做

- 不写深技术文、不投流、不做后端 / 埋点、不急着加新功能。
- 不追求 DAU / 变现。

## 7. 执行排期（建议 4 周）

原则：**你（产品向 / 写作）负责内容 + 展示 + 投递；我（助手）负责工程 + 文档初稿 + 部署。** 两者并行。

**第 1 周 · 打牢"能展示"的基础（我执行为主）**
- 部署 live demo（任务 #4）：构建并配置 GitHub Pages / Vercel，产出可访问 URL。需要你：一次性在 GitHub 启用 Pages 或授权 Vercel。
- README 加"产品故事"入口 + 顶部放 demo 链接（任务 #1）：让进仓库的人 2 分钟看到产品叙事。
- 案例文档导出 PDF（任务 #9 延伸）：一份随简历。
- 验收：有人打开链接能试用；案例 PDF 在手。

**第 2 周 · 用内容建立"产品人设"（你写作为主，我给大纲）**
- A1 竞品分析文（任务 #2）：你写 / 我给框架 → 发自己公众号 + 知乎。
- A2 产品决策文（任务 #5）：你写 / 我给框架 → 发自己公众号 + 知乎。
- 验收：2 篇产品向文章发出，带 demo 链接。

**第 3 周 · 分发与认证（你为主）**
- 提交 AlternativeTo（打 mdnice 替代品）、少数派、利器（任务 #6，我给文案）。
- V2EX「分享创造」+ 即刻：产品视角发帖（我给草稿，你发）。
- A3 公众号复盘（任务 #3）：用净排排自己文章 + 写"文科生做产品工具"（dogfooding 实证）。
- 验收：目录收录 / 在审；社区帖发出；A3 发。

**第 4 周 · 闭环与迭代（我 + 你）**
- 发布闭环 polish（任务 #7）：一键复制适配微信 + 粘贴预览，让 demo 体验完整。
- 反馈入口（任务 #8）：GitHub Issues / Discussions 说明。
- 你把"用净排发的文章数 / 反馈"补进案例文档（任务 #9 迭代）。
- 验收：demo 全链路通顺；案例有真实数据。

**持续贯穿**：你持续用净排发公众号（dogfooding 实证）；面试以案例文档 + demo 为谈资。
