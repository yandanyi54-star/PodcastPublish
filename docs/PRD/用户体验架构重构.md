# PRD — 用户体验架构重构

> 文档版本：v1.0 · 2026-07-17
> 状态：草案
> 负责人：用户体验架构师（WorkBuddy）

---

## 一、背景与目标

### 1.1 当前问题

净排自 v0.7.0 发布以来，经历了快速迭代：
- Milkdown 迁移（v0.9~）
- 深色模式（v0.10.0）
- 本地图片上传（v0.12.0）
- 装饰元素（v0.8.1）
- AI 功能（v0.6~）

但功能堆叠始终以「给现有文件加新功能」的方式进行，**从未从用户视角审视整体架构**。目前暴露出的问题：

| 问题 | 程度 | 影响范围 |
|------|------|----------|
| `App.vue` 膨胀至 3358 行 | 🔴 严重 | 所有功能维护、调试、新增 |
| 13 个工具栏图标无文字标签 | 🔴 严重 | 所有用户（尤其新用户） |
| 6 个面板系统零上下文，切换时互不相通 | 🟡 中等 | 高频用户 |
| 主题/自定义样式/品牌色三套体系用户无法理解 | 🟡 中等 | 排版调优场景 |
| 4 个 AI 按钮平铺，无场景化引导 | 🟢 较轻 | AI 用户 |
| 复制/导出双入口混淆用户 | 🟢 较轻 | 首次使用用户 |
| 新用户首屏即挫败，缺乏分步 Onboarding | 🟡 中等 | 新用户转化 |

**根因**：所有功能以「实现逻辑」而非「用户任务流」组织。工具栏按代码添加顺序排列，面板按功能模块切割，用户在不同任务间切换时需要「记住功能位置」而非「按直觉操作」。

### 1.2 目标

在**不动核心排管线（buildHtml / THEMES / callAI）**的前提下，对净排的用户体验架构进行全面重构：

1. **工具栏**从「功能导向」重新组织为「任务导向」
2. 新增**顶部上下文工具栏**（选区即操作）
3. 新增**底部状态栏**（写作反馈）
4. 设计**新手 Onboarding**（3 步卡片，降低首屏挫败）
5. 模块**组件化拆分**（为长期可维护性打基础）

### 1.3 用户场景

- **用户角色**：公众号写作者（技术/非技术）、内容编辑、自媒体运营者
- **使用场景**：每日写作 → 排版 → 复制到微信后台
- **频次**：每个公众号发布周期（日/周更新）

## 二、重构范围

### 2.1 重构概述

| 层面 | 范围 | 级别 |
|------|------|------|
| 信息架构 | 工具栏按钮排列顺序、分组方式、文字标签 | **P0** |
| 界面布局 | 新增顶部上下文工具栏、底部状态栏 | **P1** |
| 用户引导 | 新手 Onboarding 三步卡片 | **P0** |
| 模块工程 | App.vue 组件拆分（composables + 子组件） | **P1** |
| 体验细节 | 面板系统上下文保持、AI 场景化推荐 | **P2** |

### 2.2 交互流程（目标态）

**场景：首次使用**
```
打开净排 → Onboarding 三步卡片 →
  [卡片1: Markdown 写，右侧实时预览公众号样式]
  [卡片2: 选主题，一键铺满全文]
  [卡片3: 复制 HTML，粘贴微信后台]
→ 载入示例 → 沉浸写作 → 引导高亮主题切换器 → 高亮复制按钮 → 完成闭环
```

**场景：日常写作**
```
打开（自动恢复草稿）→ 沉浸写作
  → 选中文字 → 顶部浮现「AI 扩写/改写/翻译」
  → 拖入图片 → 浮现「调整宽度/替换/删除」
  → 切主题 → 预览实时更新
  → 复制 HTML → 微信粘帖
```

### 2.3 不做（边界）

- **不改动** `buildHtml.js` 排版管线（无新增 CSS 变量、无新增内联样式规则）
- **不改动** `themes.js` 主题数据结构（10 套主题字段和值全部保留）
- **不改动** `callAI.js` 调用模式（依赖注入 + AbortController 不变）
- **不改动** localStorage key 命名（`podcast_*` 前缀不动）
- **不引入**新外部依赖（Pinia、Vue Router、TypeScript 均暂不引入）
- **不改变**核心「纯前端、不联网」定位

## 三、验收标准

### P0（必须通过）

- [ ] 工具栏重排后：按钮按「写作 → 排版 → 增强 → 发布 → 系统」分组，每组有视觉分隔
- [ ] 每个工具栏按钮显示文字标签（图标 + 文字，不再仅靠 hover tooltip）
- [ ] 当前功能按钮全部保留，无任何功能丢失
- [ ] `vite build` 零错误
- [ ] 现有的 `tests/buildHtml.test.js` 和 `tests/callAI.test.js` 全部通过
- [ ] 首屏加载后编辑器正常初始化，无竞态问题

### P1（建议通过）

- [ ] 顶部上下文工具栏：选中图片时显示「调整宽度/替换/删除」
- [ ] 底部状态栏：显示字数统计 + 阅读时长预估 + 保存状态
- [ ] 模块拆分后，`App.vue` 减少至 ≤2000 行
- [ ] 拆分后各 composable 函数各由其 `.spec.js` 测试覆盖

### P2（可选）

- [ ] Onboarding 三步卡片首屏展示
- [ ] AI 面板改为上下文推荐模式

---

## 四、设计约束

- 不改动 `buildHtml` 排管线
- 不改动 `THEMES` 主题数据
- 不改动 `callAI` 业务逻辑
- 仅改动 `src/App.vue` 和新增 `src/components/`、`src/composables/` 文件
- 组件拆分原则：满足「代码可复用 || 逻辑 > 200 行 || 不增加跨文件状态管理复杂度」三项中的至少两项
